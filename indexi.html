디자인 철학은 다음과 같습니다: 단일 디자인 시스템을 유지해 사용자 혼란을 줄이고, 문서 간 이동을 즉시 할 수 있는 SPA 방식을 채택하여 서버 요청을 최소화합니다. 또한 모바일 기기에서의 가독성 확보를 위해 서체 크기와 패딩을 크게 유지했습니다.

자주 묻는 질문 (FAQ)

계정 생성 및 인증

Q: 계정 만들기는 어떻게 하나요?

A: 상단의 "회원가입" 버튼을 눌러 이메일 기반 가입을 진행하세요. 이메일 인증 후 기본 계정이 활성화됩니다. 회사/기관 계정의 경우 SSO(SAML/OAuth)를 통한 연동도 지원합니다.

파일 업로드 및 저장

Q: 파일 크기 제한은 어떻게 되나요?

A: 기본적으로 파일당 200MB로 설정되어 있지만, 관리자 설정에서 사용자 그룹별로 제한을 완화 또는 강화할 수 있습니다. 대용량 파일(>500MB)은 전용 업로드 플로우를 사용하세요.

API 관련

Q: 인증 토큰의 만료 시간은 얼마인가요?

A: 기본 액세스 토큰 만료는 1시간이며, 리프레시 토큰을 통해 재발급 가능합니다. 장기 세션이 필요한 서비스는 안전한 저장소(서버 측 비밀 저장소)를 사용하도록 권장합니다.

문제 해결 팁

  • 로그인 오류 발생 시 쿠키/세션을 삭제 후 재시도하세요.
  • 파일 업로드 실패 시 네트워크 연결 상태와 CORS 정책을 확인하세요.
  • 문제가 지속되면 운영팀에 로그(에러 메시지, 요청 ID)를 첨부해 문의해주세요.

fetch('/api/v1/assets', {headers: {Authorization: 'Bearer ' + token}})

  .catch(err => console.error('api error', err));

비밀관리와 배포 팁

오류 처리

오류 응답은 HTTP 상태 코드와 함께 표준화된 에러 객체를 반환합니다. 예:

{
  "error": {
    "code": "INVALID_TOKEN",
    "message": "Provided token is expired or invalid"
  }
}

로그 수집용 Request-ID를 헤더에 포함해 문제가 발생할 때 빠르게 추적할 수 있도록 하세요.

배포 정책

배포는 테스트→스테이징→프로덕션 순으로 진행합니다. 배포 전 체크리스트:

  • 모든 단위/통합 테스트 통과
  • 환경별 시크릿 설정 확인
  • 롤백 계획 수립(버전 태그와 데이터 마이그레이션 고려)

권장 배포 전략: 블루/그린 혹은 캔리 배포. 긴 다운타임을 피하기 위해 데이터베이스 마이그레이션은 비동기 방식으로 단계적으로 진행하세요.

접근성 가이드

기본 원칙: 모든 사용자가 서비스에 접근할 수 있도록 색 대비, 키보드 내비게이션, 스크린리더 친화적 마크업을 확보하세요.

  • 색 대비 최소 기준을 충족(AA 레벨 권장)
  • 키보드 전용 조작으로 모든 인터랙션 가능
  • ARIA 속성으로 역할과 상태 명확화

테스트 도구: Lighthouse, Axe, WAVE 등 자동 검사 도구와 실제 사용자 테스트를 병행하세요.
